Educational Reading Assessment Engine

We solve a fundamental problem: how to automatically assess oral reading fluency for children ages 5-12. Our system evaluates not just what a student reads, but how they read it — pausing, rhythm, stress, pitch, intonation, expression, and paragraph flow.

Scientific Questions

Key design decisions and research questions that shaped this system

Raw F0 values are speaker-dependent. A female teacher reading at 220 Hz and a male student reading at 120 Hz can produce identical contour patterns.

Why Contour Similarity Matters, Not Absolute Hz
Female TTS (Reference): 220 → 235 → 250 → 230 Hz Rise, Rise, Fall

similarity = max(0, 100 - abs(student_score - reference_score))

Source: comparison.py — _similarity()

F0 = Fundamental Frequency

F0 is the physical measurement of vocal fold vibration rate, measured in Hertz (Hz). It is what a computer measures.

Pitch = Perceptual Quality

Pitch is what humans perceive. It is influenced by F0 but also by intensity, duration, and speaker characteristics.

F0 vs Pitch
F0 (Physical)
  • Measured in Hz
  • Objective, repeatable
  • Speaker-dependent range
  • Children: 200-400 Hz typical
  • Adult males: 85-180 Hz
  • Adult females: 165-255 Hz
Pitch (Perceptual)
  • Subjective experience
  • Influenced by context
  • Cross-cultural variation
  • Used in musical terms
  • Not directly measurable

Our system extracts F0 using scipy autocorrelation (30ms frames, 10ms hop, 75-600 Hz range) and converts it to educational scores.

FRAME_STEP_MS = 10   # ms

Contour similarity captures the educational meaning of intonation.

The question is not "did the student hit 220 Hz?" but "did the student rise at the start and fall at the end?"

Example 1: Declarative Sentence

Expected: Falling contour at end

Reference: 250 → 240 → 230 → 220 Hz
Student A: 200 → 195 → 185 → 175 Hz

Student A: Correct contour. Both fall. Hz values irrelevant.

Example 2: Yes/No Question

Expected: Rising contour at end

Reference: 220 → 225 → 235 → 250 Hz
Student B: 150 → 155 → 165 → 180 Hz

Student B: Correct contour. Both rise.

This is implemented in intonation.py: terminal contour detection uses a 300ms window, linear regression for slope, and classifies into rising/falling/flat/uncertain.

Core Metrics

Each metric has a specific educational purpose, verified formula, and clear limitations

What is this metric?

Measures the quality and appropriateness of inter-word silence duration during reading.

Why do we use this metric?

Pausing demonstrates comprehension and natural phrasing. Children who pause appropriately show better text understanding.

Formula (Linguistic Pause Model)

pause_score = 0.60 × threshold + 0.25 × consistency + 0.15 × variation

Scaled to 0-100. Each pause is scored against context-appropriate thresholds.

Thresholds (ms)

ContextMin (ms)Max (ms)
Sentence-internal80400
Sentence boundary2501200
Inter-word100800
Chunk boundary2001500

Source: config.py — SENTENCE_INTERNAL_*, SENTENCE_BOUNDARY_*, INTER_WORD_*, CHUNK_BOUNDARY_*

Worked Example

Input: 10 words, 9 gaps, 7 in optimal range

threshold: 7/9 = 0.78

consistency: 0.78 (CV within range)

variation: 0.80 (moderate CV 0.2-0.6)

Result: 0.60 × 0.78 + 0.25 × 0.78 + 0.15 × 0.80 = 78/100

Score Interpretation

80-100: Excellent — natural, comprehension-indicating pauses 60-79: Good — appropriate pausing with minor issues 0-39: Poor — pausing patterns suggest reading difficulty

Limitations

  • Single-word chunks return null (confidence = 0)
  • Chunks with no pauses return 100 (confidence = 1.0) — this is correct when pause_count = 0 and word_count > 1
  • Thresholds are calibrated for ages 5-12, US English

What is this metric?

Measures the regularity and consistency of speech timing during reading.

Why do we use this metric?

Regular rhythm indicates fluent reading with consistent pacing. Irregular rhythm suggests decoding difficulty.

Formula

cv = stdev(intervals) / mean(intervals)
rhythm_score = stability(cv)  # piecewise mapping

Piecewise Stability Mapping

CV RangeScore RangeMeaning
CV ≤ 0.01.0Perfect (identical intervals)
0.0 < CV < 0.10.85 - 1.0Excellent stability
0.1 ≤ CV ≤ 0.30.6 - 0.85Good stability
0.3 < CV ≤ 0.50.4 - 0.75Moderate variation
0.5 < CV ≤ 0.80.2 - 0.55High variation
CV > 0.80.0 - 0.2Very irregular

Source: rhythm_stress.py — _score_rhythm_stability()

Worked Example

Input: Intervals [320, 340, 310, 350, 330] ms

Mean: 330 ms, StDev: 15.8 ms

CV: 15.8 / 330 = 0.048

Result: CV < 0.1 → score = 100/100

Limitations

  • Requires at least 5 words (RHYTHM_MIN_WORDS = 5)
  • Fewer than 5 words: rhythm_score = None, confidence = 0.0
  • Confidence scales with word count: confidence = min(1.0, intervals / 3.0)

What is this metric?

3D prominence model: measures word emphasis using duration, F0, and energy.

Formula (3D Model)

stress = 0.40 × duration_prominence + 0.30 × f0_prominence + 0.30 × energy_prominence

When F0 or energy unavailable, weights redistribute: 55% duration + 45% available metric.

Worked Example

Input:

Duration prominence = 0.80

F0 prominence = 0.70

Energy prominence = 0.90

Calculation: 0.40 × 0.80 + 0.30 × 0.70 + 0.30 × 0.90

Result: 0.32 + 0.21 + 0.27 = 0.80 = 80/100

How Each Dimension Works

DimensionMethodWeight
DurationSyllable-normalized duration variance. Words > 1.2x mean are stressed.40%
F0Peak F0 prominence. Words > 1.1x baseline are stressed, < 0.9x unstressed.30%
EnergyRMS energy prominence. Same threshold logic as F0.30%

Source: rhythm_stress.py — compute_stress_metrics(), _compute_f0_prominence(), _compute_energy_prominence()

Limitations

  • F0 and energy extraction may fail (non-fatal fallback)
  • Syllable counting uses vowel-group heuristic (~85-90% accuracy)
  • Stress is relative within the utterance, not absolute

What is this metric?

Speech rate and pause ratio analysis. Computed but NOT included in any scoring formula.

Status

Diagnostic Only — Not in Scoring Weights

Fluency is computed in pause_fluency.py and returned in the response, but is NOT part of CHUNK_WEIGHTS, EXPRESSION_WEIGHTS, or PARAGRAPH_PROSODY_WEIGHTS.

Formula

speech_rate = word_count / total_duration
pause_ratio = pause_time / total_duration
fluency_score = (rate_score + pause_score) / 2.0

Rate scoring: 1-4 words/sec = optimal. Pause ratio: 5-30% = optimal.

Why Not in Scoring?

Fluency overlaps significantly with Pause and Rhythm. Including it would double-count pause information. It remains available as a diagnostic field in the response.

What is this metric?

F0 (fundamental frequency) variability and range scoring. Measures expressiveness of pitch movement.

Pitch vs F0

F0 is the physical measurement (Hz). Pitch is the perceptual quality. Our "Pitch" score is an educational interpretation of F0 variability.

Formula

pitch_score = (range_score × 0.5 + var_score × 0.5) × 100

Scoring Thresholds

ComponentOptimal RangeBelow OptimalAbove Optimal
F0 Range (Hz)20 - 100 Hz → 1.0< 20 Hz → linear to 0> 100 Hz → linear to 0
F0 Variability (CV)0.02 - 0.15 → 1.0< 0.02 → linear to 0> 0.15 → linear to 0

Source: f0_extraction.py — score_pitch_expressiveness()

Worked Example

Input: F0 range = 45 Hz, Variability = 0.08

range_score: 45 is in [20, 100] → 1.0

var_score: 0.08 is in [0.02, 0.15] → 1.0

Result: (1.0 × 0.5 + 1.0 × 0.5) × 100 = 100/100

Known Limitation

Absolute Hz thresholds penalize children. Children naturally have wider F0 ranges (200-400 Hz). The threshold of 100 Hz optimal range means a child with 150 Hz range gets a reduced score. This is a known issue pending classroom calibration.

What is this metric?

Composite expression score combining intonation, stress, pause variation, and rhythm.

Formula

expression = 0.35 × intonation + 0.30 × stress + 0.20 × pause_variation + 0.15 × rhythm

Fluency is NOT included. Each component is confidence-gated (MIN_CONFIDENCE = 0.3).

Worked Example

Input: Intonation: 70, Stress: 65, Pause Variation: 75, Rhythm: 85

Calculation: 0.35 × 70 + 0.30 × 65 + 0.20 × 75 + 0.15 × 85

Result: 24.5 + 19.5 + 15.0 + 12.75 = 72/100

Confidence Gating

Components with confidence < 0.3 are excluded. Remaining weights are renormalized. If no component meets the threshold, expression = 0.0.

Source: expression.py — compute_expression_metrics()

What is this metric?

Sentence-to-sentence consistency (paragraph_flow) and speech-rate consistency (story_flow).

Formula

paragraph_overall = 0.60 × paragraph_flow + 0.40 × story_flow

How Each Component Works

ComponentMethodScore Formula
paragraph_flowSentence overall score consistencyscore = max(0, 100 - CV × 200)
story_flowSpeech-rate consistencyscore = max(0, 100 - CV × 200)

Source: level_scoring.py — _paragraph_flow(), _story_flow(), _consistency()

Consistency Scoring

cv = pstdev(values) / mean(values)
score = max(0, 100 - cv × 200)
# cv = 0 → score = 100
# cv = 0.5 → score = 0

Limitations

  • Requires at least 2 sentences (returns null with fewer)
  • Recording 3 only
  • Confidence = min(1.0, sentence_count / 3)

Tool Deep Dives

What Praat, OpenSMILE, and Librosa actually do

Praat

Research-Grade

What is Praat?

Praat (Dutch for "talk") is a free scientific software package for phonetic analysis. Developed by Paul Boersma and David Weenink at the University of Amsterdam since 1992. Over 30 years of academic validation. The gold standard for acoustic phonetics research.

What Praat Measures

F0 Tracking Autocorrelation, cross-correlation, subharmonic-to-harmonic ratio methods
Intensity dB contours, intensity range, intensity contour analysis
Formants F1-F5 tracking, vowel space analysis
Voice Quality Jitter, shimmer, harmonics-to-noise ratio
Durations Phoneme, syllable, word duration measurement
Spectrograms Wideband, narrowband, mel-frequency spectrograms

Praat Example: "The cat sat"

Mean F0: 220 Hz

Intensity: 72 dB

Duration: 1.8 seconds

F0 Range: 180-250 Hz

What Praat CANNOT Provide

  • Fluency scoring
  • Expression assessment
  • Teacher explanations
  • Recording 1/2/3 progression
  • Educational interpretation
  • Real-time production deployment
  • 0-100 scoring system

OpenSMILE

Affective Computing

What is OpenSMILE?

An open-source toolkit for large-scale audio feature extraction. Primarily designed for emotion recognition and affective computing. Created by Florian Eyben, Björn Schuller, and others. The C++/Python library extracts thousands of acoustic features.

Feature Sets

eGeMAPS 88 features — extended Geneva Minimalistic Acoustic Parameter Set. Designed for affective computing research.
ComParE 6,373 features — Computational Paralinguistics Challenge feature set. Comprehensive but overwhelming.
Emotion Recognition Happy, sad, angry, neutral, fear, disgust, surprise classification.

Trade-offs

Strengths
  • 6,373+ acoustic features in one pass
  • Battle-tested emotion recognition
  • Well-documented feature definitions
  • Supports real-time processing
Weaknesses
  • 6,373 features is overkill for reading assessment
  • Designed for emotion, not education
  • No teacher-friendly output format
  • Feature selection burden falls on us

Librosa

Audio Analysis

What is Librosa?

A Python package for music and audio analysis. Provides spectral, rhythmic, and temporal features. Primarily designed for music information retrieval, not speech prosody.

Strengths

  • Excellent for music analysis (tempo, beat, harmony)
  • Good spectral features (MFCC, chroma, mel spectrograms)
  • Clean Python API, well-documented
  • Strong visualization tools

Weaknesses for Speech Prosody

  • No F0 extraction (that is Praat's domain)
  • No phoneme-level analysis
  • No educational scoring framework
  • Music-centric, not speech-centric
  • No real-time production deployment model

Verdict

Librosa is not relevant for this project. It is an audio analysis toolkit, not a speech prosody framework. Useful for music applications, not educational reading assessment.
